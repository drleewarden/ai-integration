# AI Implementation Roadmap — 90-Day Template

**Creative Milk — Members Library · v1 (July 2026)**

This is the phased rollout plan we use with consulting clients, stripped to a template you can adapt in an afternoon. The companion file `roadmap-tracker.csv` opens in Excel or Google Sheets — it's the same plan as a working tracker with owner, effort and status columns.

**How to use it:** read the principles, then work through the phases replacing the bracketed examples with your own. Delete anything that doesn't apply. A roadmap you'll actually follow beats a comprehensive one you won't.

---

## Three principles before you start

1. **One workflow at a time.** Every failed AI rollout we've seen tried to change five things at once. Every successful one shipped one workflow, banked the win, then moved.
2. **Pick by payback, not by excitement.** Your first automation should be the one with the clearest hours-saved number, not the most impressive demo. (The Automation Opportunity Finder and Cost-Benefit Workbench in the members library exist for exactly this.)
3. **Name an owner.** Not "the team". One person owns the rollout, even in a two-person business. If it's everyone's job, it's no one's.

## Phase 0 — Foundations (Days 1–10)

*Goal: rules and a baseline, so everything after this is safe and measurable.*

| # | Milestone | Owner | Effort | Done when |
|---|---|---|---|---|
| 0.1 | AI usage policy agreed and shared | [Owner] | 2 hrs | Everyone who uses AI tools has read it |
| 0.2 | Approved tools list written (tool, plan, purpose) | [Owner] | 1 hr | List exists and is where people can find it |
| 0.3 | Data safety rules adopted (see Data Safety Checklist) | [Owner] | 1 hr | De-identify rule + payment-details rule agreed |
| 0.4 | Baseline numbers recorded | [Owner] | 2 hrs | Hours/week on target tasks written down, honestly |

**Baseline matters most.** Write down today's numbers for the workflows you plan to touch — hours spent, response times, error rates. Without a baseline, day 90 is opinions.

## Phase 1 — First win (Days 11–35)

*Goal: one workflow automated end-to-end, measured, and visibly working.*

| # | Milestone | Owner | Effort | Done when |
|---|---|---|---|---|
| 1.1 | Target workflow chosen by payback | [Owner] | 1 hr | One workflow named, e.g. [lead follow-up] |
| 1.2 | Success metric agreed | [Owner] | 30 min | e.g. "every enquiry answered within 1 minute" |
| 1.3 | Tool selected and configured | [Owner] | 0.5–2 days | Test run passes with dummy data |
| 1.4 | Live with human review | [Owner] | 1 week elapsed | Running on real work, outputs checked daily |
| 1.5 | Review vs baseline; keep, fix or kill | [Owner] | 1 hr | Decision recorded with the numbers |

**The human-review week is not optional.** Every automation runs supervised before it runs alone. Most problems surface in the first twenty real cases.

## Risk checklist

Review at each phase boundary. Tick = handled.

- [ ] **Data:** nothing on the "never paste" list is reaching AI tools (spot-check, don't assume)
- [ ] **Quality:** every automated output has a defined review step or a human path
- [ ] **Dependency:** if the AI tool disappeared tomorrow, we could run the workflow manually within a day
- [ ] **Customers:** anywhere AI talks to customers, a human is one step away and the tone has been checked
- [ ] **Staff:** the rollout has been framed as removing drudge work, and the team's actual concerns have been asked about, not assumed
- [ ] **Spend:** total AI subscription cost is known and reviewed against the baseline savings
- [ ] **Compliance:** if you're in a regulated space (health, finance, legal, childcare), your obligations have been checked against each new workflow

---

## The day-90 question

Would you sign up for the last 90 days again at twice the effort? If yes — the roadmap worked; run it again on the next workflow. If no — the honest review in 3.1 will tell you whether the problem was the tool, the workflow chosen, or the time invested. All three are fixable; pretending isn't.
